# 📝 代码规范

## 1. 文件命名

统一格式：`题目编号_题目名称.py`

示例：`1.两数之和.py`、`49.字母异位词分组.py`

## 2. 代码注释要求

每道题的代码需包含以下注释：

- 解题思路简述

- 关键步骤注释

- 复杂度分析

示例：

```python

# 题目链接：https://leetcode.cn/problems/two-sum/
# 解题思路：使用哈希表存储已遍历元素的「值-下标」映射，遍历过程中查找补数
# 时间复杂度：O(n)，遍历数组一次，哈希表操作均为O(1)
# 空间复杂度：O(n)，最坏情况下存储n-1个元素

from typing import List

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        num_map = {}  # key: 数字，value: 下标
        for idx, num in enumerate(nums):
            complement = target - num
            if complement in num_map:
                return [num_map[complement], idx]
            num_map[num] = idx
        return []
```

# 💡 刷题技巧汇总

## 1. 通用技巧

- 拿到题目先分析题干，明确输入输出约束，列举边界情况（如空输入、极值输入）

- 优先想暴力解法，再思考如何优化（常用优化方向：哈希表降维、双指针、二分查找）

- 写代码前先梳理逻辑，必要时画流程图或思维导图

- 完成代码后，用至少3组测试用例验证（正常用例、边界用例、异常用例）

## 2. 题型专属技巧

- **数组/链表**：双指针（快慢指针、左右指针）是高频优化手段，适用于去重、找最值、反转等场景

- **哈希表**：核心用于「快速查找」和「分组统计」，Key的设计是关键（如字符计数、排序字符串）

- **二叉树**：优先考虑递归（前中后序遍历），非递归遍历需借助栈/队列，层序遍历用BFS

- **动态规划**：先定义DP状态，再推导转移方程，最后确定初始条件和边界，可先画DP表辅助分析
